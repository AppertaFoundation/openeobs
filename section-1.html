General Styling

Style components used for main UI

Modifiers

.header

Top level menu container

.content

Main content, is offset to accommodate top level menu

.content.no-menu

Restores the offset as menu isn't present

.share-content

Adds another offset to the bottom to accommodate for the stand-in screens floating footer

.footer

Footer, contains logged in user details

.footer .user

User details in footer

.share-footer

Container for footer used in stand-in screen

.share-footer .user

User details in stand-in screen

Colour Palette

Colours used in Open-eObs

Modifiers

@header-colour

282828, Used in the header as a less harsh tone of black

@body-colour

000000, Used in main body for high contrast

@action-colour

ea3172, Main colour from Open-eObs branding, used as a call to action

@link-colour

4491ce, Secondary colour from Open-eObs branding, used as a dismissive action

@error-color

red, Used in a less harsh form throughout the app via a LESS transformation, Used to dictate high clinical risk

@warning-colour

ffd5b4, Used to dictate medium clinical risk

@ok-colour

green, Used in a less hard form throughout the app via a LESS transformation, Used to dictate low clinical risk

@not-set-colour

e5dfff, Used to dictate an unknown clinical risk

Iconography

Icons used to display information

Modifiers

i

Element used for icons

.icon-info

A 'i' in a circle, used for patient information

.icon-alert

An exclamation mark in a triangle, used for alerts

.icon-left-arrow

A left pointing arrow, used to indicate a clinical risk trend that hasn't changed

.icon-same-arrow

An alias for the icon-left-arrow class

.icon-up-arrow

An upwards pointing arrow, used to indicate a clinical risk that has increased

.icon-down-arrow

A downwards pointing arrow, used to indicate a clinical risk that has decreased
